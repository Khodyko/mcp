Корнем всех вызовов является класс SupportAssistantController.

В нем описана работа с:
1) Оркестраторами тулов.
    simpleOrchestrator (отдает решение о выборе тулов llm)
    aiBasedOrchestrator  (сужает выбор тулов с помощью дополнительно запроса в llm)
    toolBasedOrchestrator (делает тоже самое, что и прошлый, но через tool)
Все эти оркестраторы выполняют 3 функции через тулы:
1. Загрузка в бд пользователей из json файла (DocumentLoaderAgent)
2. Загрузка в бд и векторное хранилище продуктов из json файла (DocumentLoaderAgent)
3. Получение описания продукта (ProductAgent)
4. Заказ продукта (OrderAgent)

Пункты 2, 3, 4 вместе - это Rag, когда это выполняется через llm.
Однако применение в данном проекте не совсем стандартное, так как все вышеперечисленные классы вызываются именно как тулы.
Если вам нужно построить проект RAG без tool, стоит посмотреть другой пример. 
Этот пример показывает более глубоко возможности tool.
2) С различными воркфлоу. (https://spring.io/blog/2025/01/21/spring-ai-agentic-patterns)
   SimpleAiChain
Показывает как можно работать с вызовами llm.
